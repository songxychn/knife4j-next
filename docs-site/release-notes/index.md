# 发布说明

这个页面先服务一个现实目标：让社区明确知道 `knife4j-next` 已经恢复了“可发布、可说明、可追踪”的节奏。

## 当前版本

### 4.6.0.1

- Maven `groupId` 已切到 `com.baizhukui`
- 仓库已经补齐 GitHub Actions 构建与发布流程
- 当前优先处理兼容性、回归问题和发版恢复

## 这一页后续应该承接什么

- 每个版本修复了哪些兼容性问题
- 哪些问题仍在处理中
- 哪些内容只是内部准备，还没有正式发布承诺

## 为什么这个页面重要

如果你想让用户相信“这是可迁移的社区继任者”，那发布说明就不能再只是 tag 列表。它需要帮用户判断：

- 现在这个版本值不值得升
- 我关心的问题有没有被覆盖
- 这个项目是不是还在活跃维护
