# 自治权限策略

本文件定义自治 agent 在无人值守时可以做什么。

## 无需额外批准即可执行

- 创建或更新文档
- 细化 `.agent/` 状态文件
- 修复范围清晰且有验证的窄范围 bug
- 增加或改进定向测试
- 清理意图明确的陈旧表述
- 创建或更新 PR
- 发现新工作时创建后续任务

## 仅当任务明确授权时才可执行

- 修改 CI workflow 逻辑
- 修改依赖版本
- 引入新模块或 package
- 更新生成物
- 改变 `knife4j-front/knife4j-ui-react` 的公开行为

## 永远不要自治执行

- 发布 release
- 直接 push 到 `main`
- 修改 secrets、tokens 或凭据接线
- 修改 Maven 坐标或默认兼容性承诺
- 删除模块、大目录或重要历史代码路径
- 代表维护者合并 PR
- 执行破坏性 git 操作来掩盖问题

## 安全任务形态

只有同时满足以下条件的任务，才适合自治执行：

- 有一个清晰的责任区域
- 有客观验证命令
- 回滚路径简单
- 不需要产品策略判断

## 升级触发条件

遇到以下情况时升级给维护者：

- 两个合理修复方案会产生不同外部行为
- 测试在当前任务范围之外失败
- 任务需要跨模块架构判断
- 改动影响 release、兼容性或支持承诺
- agent 需要猜测用户意图

## 推荐行为

- 偏向小 PR
- 为下一次运行留下清晰记录
- 遇到含糊时干净停止，而不是临场发挥
- 当探索或实现会消耗大量上下文时，委派给短命 worker，保持 coordinator 上下文较小
