# Agent 权限边界

维护者在场时的默认权限。拿不准就问维护者，不要猜。

## 可以直接做

- 文档与注释的澄清、错别字、过时表述清理
- 范围清晰且有验证的窄范围 bugfix
- 增加或改进定向测试
- 创建 / 更新 PR
- 用 issue 记录后续工作

## 需要任务或维护者明确授权

- 改 CI workflow 逻辑
- 改依赖主版本或大范围升级
- 引入新模块 / package
- 更新生成物
- 改变 `front/ui-react` 的对外行为或产品承诺

## 永远不要擅自做

- 发布 release、推送 release tag
- 直接 push `master`
- 改 secrets、tokens 或 release 凭据接线
- 改 Maven 坐标或默认兼容承诺
- 删除模块、大目录或重要历史代码路径
- 代替维护者合并 PR
- 用破坏性 git 操作掩盖问题（`--no-verify`、乱 force-push 等）

## 适合 agent 推进的任务

同时满足：

- 责任区域清晰
- 有客观验证命令
- 回滚简单
- 不需要产品策略判断

## 必须停下来问维护者

- 两个合理方案会产生不同外部行为
- 失败超出当前任务范围
- 需要跨模块架构取舍
- 影响 release、兼容性或支持承诺
- 只能靠猜测用户意图才能继续
